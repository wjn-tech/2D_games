## Context
米娜终局战目前核心特征是“镜像玩家生命与法杖”，但对玩家行为层的干预不足，终局识别度主要依赖数值与弹道压力。本变更引入控制干扰与分段压制机制，目标是在不破坏现有场景隔离和快照恢复契约的前提下，提升终局战术层次。

## Goals / Non-Goals
- Goals:
  - 为米娜提供可读、可反制、可验证的控制类机制。
  - 将“生命阈值推进”与“玩家输出受限”绑定，形成终局节奏曲线。
  - 保证所有临时状态在遭遇结束后可完整回滚。
- Non-Goals:
  - 不重构全局输入系统。
  - 不修改三王（史莱姆/骷髅/魔眼）的行为合同。
  - 不引入新的全局持久化字段存储临时 Debuff。

## Decisions
- Decision: 引入“机制窗口”统一调度米娜控制技能，而非每帧概率触发。
  - Why: 便于平衡与验证，避免连续不可读控制链。
- Decision: 米娜增加双来源“无敌 + 投射物吞噬”窗口（定时 10s 与血量阈值 20%）。
  - Why: 通过可预期周期与阶段性爆发共同塑造终局节奏。
- Decision: 控制效果统一定义持续时间与恢复动作，全部归于遭遇态临时状态容器。
  - Why: 降低状态泄漏风险，确保胜败结算后一致恢复。
- Decision: 玩家攻击衰减采用阈值触发 + 叠乘下限（最小 0.2）。
  - Why: 满足“每 1/5 血降低 20%”意图，同时避免 0 伤害死锁。
- Decision: 投射物吞噬成长采用“每吞噬 1 发，米娜伤害倍率 ×1.01”，且仅在当前遭遇有效。
  - Why: 直接响应玩家投射密度，形成战术权衡并避免跨遭遇永久膨胀。
- Decision: 输入颠倒默认仅作用于战斗移动轴，不修改 UI/系统快捷键。
  - Why: 保留玩法压制同时避免可用性灾难。
- Decision: 位置互换必须执行安全落点校验，失败时降级为“仅米娜短距位移”。
  - Why: 防止嵌墙/越界导致战斗中断。
- Decision: 定时无敌与阈值无敌重叠时采用“结束时间取最大值”的窗口合并策略。
  - Why: 避免状态竞争造成窗口抖动或错误提前结束。

## Risks / Trade-offs
- 风险: 多个控制效果叠加可能造成玩家挫败感过高。
  - Mitigation: 技能互斥窗口、触发冷却、前摇提示和短暂豁免。
- 风险: 临时状态与既有状态系统（如中毒、重力控制）冲突。
  - Mitigation: 定义状态优先级与回滚顺序，加入中断恢复验证。
- 风险: 阈值降攻与法杖构筑叠加后难以平衡。
  - Mitigation: 设置倍率下限并在提案阶段约束为可调参数。
- 风险: 高频投射构筑可能导致米娜伤害成长过快。
  - Mitigation: 用清晰反馈呈现成长过程，并将成长限定在单次遭遇内。
- 风险: 吞噬逻辑与既有投射物系统碰撞筛选冲突。
  - Mitigation: 在验证中覆盖定时窗口与阈值窗口两条吞噬路径及边界条件。

## Migration Plan
1. 先落契约：明确六类机制触发条件、持续时间、恢复规则。
2. 再落阈值：接入米娜生命阈值事件与玩家攻击衰减、阈值无敌窗口。
3. 再落吞噬：接入定时吞噬与吞噬计数伤害成长。
4. 再落反馈：HUD/浮字/提示整合，补齐可读性。
5. 最后回归：胜败/中断恢复、机制互斥、边界安全验证。

## Verification Strategy
- 机制正确性：逐项验证六类机制触发、持续与恢复。
- 状态隔离：验证遭遇结束后玩家状态与输入完全复原。
- 数值推进：验证 20% 阈值触发次数、顺序、倍率下限。
- 吞噬推进：验证投射物吞噬计数、伤害成长、遭遇结束清零。
- 稳定性：验证位置互换的安全落点与失败降级路径。

## Implementation Details (Applied)
- 米娜机制调度：`src/systems/boss/mina_finale.gd`
  - 新增控制技能窗口、每 10 秒无敌窗口、20% 血量阈值事件、投射物吞噬与吞噬增伤。
  - 新增机制可用性筛选（按效果冷却与玩家当前受控状态筛掉不可用效果），避免同类控制无意义重叠刷新。
  - 新增机制反馈浮字：空间扰动、心绞痛、禁射、输入颠倒、重力翻转、无敌窗口来源提示与吞噬成长提示。
  - 新增无敌判定覆盖 `take_damage`，并实现窗口重叠按剩余时长最大值合并。
- 玩家受控状态：`scenes/player.gd`
  - 新增心绞痛、禁射、输入颠倒、重力翻转、攻击衰减接口与计时回滚。
  - 在移动、施法与重力流程中接入状态影响，并提供遭遇结束统一清理方法。
  - 新增公开状态查询接口（投射锁/输入颠倒/重力翻转/心绞痛）供米娜技能调度读取当前互斥状态。
- 伤害倍率管线：`src/systems/magic/spell_processor.gd` 与 `src/systems/boss/boss_base.gd`
  - 法术弹体与 Boss 近战接触均接入 `get_combat_damage_multiplier` 钩子。
  - 玩家阈值降攻与米娜吞噬增伤统一通过该倍率管线生效。
- 投射物吞噬识别：`src/systems/magic/projectiles/projectile_base.gd`、`src/systems/boss/boss_projectile.gd`
  - 分别新增分组标记 `magic_projectiles` 与 `boss_projectiles`，供米娜吞噬扫描。
- 状态回滚保障：`src/systems/boss/boss_encounter_manager.gd`
  - 在遭遇入场与结算路径调用玩家状态清理接口，避免米娜临时 Debuff 泄漏到主世界。
