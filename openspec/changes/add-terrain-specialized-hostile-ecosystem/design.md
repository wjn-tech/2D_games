## Context
项目已经具备严格地形匹配的刷怪判定维度，但内容层仍是“小池+弱差异”。本次变更目标不是再造生成系统，而是在现有框架上构建“31地形-15家族”的可维护生态网络。

输入来源：
- 用户提供的15种怪物概念（含地形归属与战斗机制）。
- 现有地图语义：map_biomes、depth_bands、cave_regions、underworld_regions。

## Goals / Non-Goals
- Goals:
  - 在不破坏现有世界生成与刷怪管线的前提下，提供地形专属怪物生态。
  - 将怪物设计转换为标准化生成数据与签名战斗行为要求。
  - 提供可验证的覆盖率与平衡基线。
- Non-Goals:
  - 本提案不直接产出最终美术质量与完整动画资产（占位资源先行）。
  - 本提案不引入理智、口渴、装备耐久腐蚀、QTE 挣脱系统。

## Data Model Decisions
1. 生成表保持“显式严格匹配”，不依赖隐式默认值。
2. 新增可选扩展字段（提案阶段）：
   - terrain_priority: 同一规则命中时用于冲突解决。
   - hotspot_multiplier: 特定地形热点倍率（用于拟态集群和专精怪）。
   - rarity_tier: common/uncommon/rare/elite。
   - behavior_profile_id: 绑定签名攻击包。
3. 所有规则必须可追溯到地形语义维度，禁止纯全局随机规则。

## Architecture Decisions
1. 将变更拆分为两个能力：
   - hostile-terrain-spawning: 负责“在哪刷、刷多少、是否公平”。
   - hostile-signature-behaviors: 负责“刷出来后如何战斗、如何被反制”。
2. 采用分阶段交付而非一次性大爆炸：
   - 阶段A先打通规则与验证，再上内容，避免大规模返工。
3. 高风险机制默认开启：
  - 即死、强控类机制默认上线，后续通过参数平衡而非默认关闭控制体验。

## Balance and Validation Strategy
1. 覆盖率：每个地形至少2个候选敌对家族。
2. 压力约束：同屏上限、局部密度、峰值并发与帧耗预算。
3. 公平性：生成点可读、可规避、可反制，不允许无前摇硬判定伤害链。
4. 种子一致性：同seed同条件下刷新分布统计应在容忍误差内稳定。
5. 稀有全域怪：拟态集群基础概率 0.12%，口袋洞/连通道/实心岩层倍率 x2。

## Risks / Trade-offs
- 风险：机制复杂度高，导致学习成本与挫败感上升。
  - 缓解：先上“基础行为+轻机制”，逐步开启重机制。
- 风险：31地形映射维护成本高。
  - 缓解：提供地形覆盖报告与规则完整性校验。
- 风险：新增状态系统耦合战斗与UI。
  - 缓解：状态系统按依赖分层，未就绪时降级为占位效果。
