## Context
- 现有小地图已支持点击展开，但缺少滚轮与按钮缩放入口。
- 小地图当前以玩家位置为中心显示，缺少玩家自定义标记功能。
- 时间/天气信息虽已有数据源，但与小地图区域尚未形成统一信息卡。

## Goals / Non-Goals
- Goals:
- 让三种缩放交互并存且行为一致。
- 增加玩家自定义标记点能力。
- 在小地图下方提供仅含时间/天气的卡片信息。
- 适配 16:9、16:10、21:9。
- Non-Goals:
- 不引入 NPC/资源自动标记。
- 不改天气世界渲染与地形细节。
- 不使用 HTML 壳。

## Decisions
- Decision: 延续现有 MinimapUI 控制脚本，增量扩展输入与标记层，不重写整套地图系统。
- Decision: 三种缩放入口共享同一 zoom 状态与上下限，避免行为分裂。
- Decision: 标记点仅为玩家自定义，先满足最小闭环，后续再评估持久化与分类。

## Risks / Trade-offs
- 风险: 多输入通道并存可能产生冲突。
- Mitigation: 定义输入优先级与 UI 焦点保护，统一由 MinimapUI 处理。
- 风险: 标记层在缩放后出现偏移。
- Mitigation: 统一使用地图归一化坐标存储与渲染换算。

## Migration Plan
1. 先统一 zoom 状态机与按钮/滚轮/点击接入。
2. 再接入玩家自定义标记层。
3. 最后增加时间天气卡片并做响应式回归。

## Open Questions
- 标记持久化暂不纳入本小提案，后续可单独提案扩展。
