## Context
当前运行时液体逻辑具备 active-cell 调度、下落冷却、分包下落与局部侧向扩散能力，但仍会在以下情境出现长期异常：
- 大体积水体内部空泡长期存在。
- 边界或高负载条件下出现反重力悬滴观感。
- 阈值与补丁逻辑互相耦合，调参后易回归。

该 change 的目标不是再添加更多修复分支，而是重建“主循环天然收敛”的核心合同。

## Goals / Non-Goals
- Goals:
  - 主逻辑在无修复 pass 前提下消除长期空泡与长期悬滴。
  - 保证跨区块与卸载流程下的质量守恒。
  - 在预算受限情况下保证最终可处理性（eventual progress）。
  - 明确渲染阈值与物理连通性的一致性边界。
- Non-Goals:
  - 不引入全局刚体液体或 screen-space 权威模拟。
  - 不在本 change 内扩展新液体类型玩法矩阵（如 Honey/Shimmer 完整规则）。
  - 不重做美术风格，仅定义显示一致性合同。

## Decisions
- Decision: 用“收敛不变量”替代“补丁 pass 组合”作为核心设计轴。
  - Invariants:
    - 质量守恒：任何转移、跨区块交接、区块卸载都不得吞液。
    - 单元边界：每格液量始终满足 0 <= amount <= 1。
    - 下行可达单调性：若存在可达下行路径，源格在有限 tick 内出现净下降。
    - 有界均压：同层连通区压差在有限 tick 内衰减，不产生永久高低台阶。

- Decision: 将跨区块流动改为“可交接”而非“不可处理就悬挂”。
  - 对下方区块未就绪情形，定义显式边界交接状态（pending handoff），并要求在目标区块就绪后继续沉降。
  - 禁止以“简单跳过更新”替代交接语义。

- Decision: 调度层采用公平队列与冷却到期激活，提供最终可处理保证。
  - 在预算受限时可延后，但不得永久饥饿。
  - 需要可验证的 starvation 上限或统计告警指标。

- Decision: 渲染一致性采用双阈值合同。
  - 逻辑最小残量阈值与可见阈值需满足一致性约束，避免“逻辑连通但视觉断裂”的悬浮错觉。
  - 当细膜承担连通责任时，渲染层必须提供可感知表现（不要求全量高亮，但不能完全消隐）。

## Risks / Trade-offs
- 收敛性增强可能降低局部吞吐峰值，需要在视觉连续性与总排空时间之间折中。
- 跨区块交接状态会增加状态机复杂度，需要明确持久化语义避免保存回归。
- 公平调度若无预算护栏，可能抬升帧耗；需配套预算分级与退化策略。

## Migration Plan
1. 先落地规格与验证场景，冻结收敛性验收标准。
2. 再实现核心求解与边界交接重构，保持外部 API 尽量不变。
3. 引入阶段性指标（空泡存活时长、悬滴存活时长、质量误差）进行回归门禁。
4. 如出现性能回退，优先调度预算分级，不回退收敛合同。

## Open Questions
- 跨区块 pending handoff 是写入 chunk runtime state，还是单独维护短生命周期 transfer log。
- 视觉连通最低可见策略采用“细膜增强”还是“连接高亮”，需要最终美术确认。
