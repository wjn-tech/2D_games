## Context
当前项目液体系统已经具备活跃单元队列、冷却调度、分块持久化和覆盖层渲染，但近期持续出现“空泡不收敛、悬滴反直觉、边界场景不稳定、高负载抖动”四类问题。历史修复主要通过阈值与补丁式 pass 叠加完成，维护复杂度正在上升。

本次变更目标不是继续调参数，而是建立新的权威运行时架构，使正确性、性能和视觉一致性由主路径保证。

## Goals
- 建立可证明的守恒与收敛合同：
  - 质量守恒
  - 垂向可达优先
  - 有界均压收敛
- 建立可预算的调度模型：高活跃快更新、低活跃慢更新、无永久饥饿。
- 建立跨区块连续语义：边界不可写时不丢液，恢复后可继续演化。
- 建立物理-渲染一致性：减少视觉浮空错觉。

## Non-Goals
- 本次不引入 Navier-Stokes 连续流体求解。
- 本次不追求真实湍流、涡量、表面张力等高阶物理。
- 本次不扩展新液体家族玩法（蜂蜜/特殊反应液体留到后续）。

## Key Decisions
### Decision 1: 采用固定精度格子质量作为权威状态
- 每格维护固定精度液体量（例如 0..255）和液体类型。
- 以固定精度替代浮点作为主状态，减少小数漂移和阈值抖动。
- 需要时在渲染层做连续化插值，不影响权威状态。

Rationale:
- 便于做质量守恒审计。
- 与 Terraria 风格 tile liquid 表达一致，工程上更稳。

### Decision 2: 主求解路径内建“空泡收敛”而非后置修补
- 核心顺序为：
  - 垂向可行流动
  - 侧向均压
  - 受限上返压（仅在满足压差条件时）
- 后置修补 pass 仅作为保险丝，不承担主正确性。

Rationale:
- 避免“主逻辑错误 + 后处理掩盖”的长期维护风险。

### Decision 3: 调度采用公平队列 + 时间轮
- 活跃队列只处理热点单元。
- 冷却与延迟重试由时间轮管理，避免每帧扫描全体 key。
- 邻接单元更新频率差异受限，避免高频单元把低频邻居“饿死”。

Rationale:
- 借鉴 CA 动态更新率思想，将预算集中在变化区域。

### Decision 4: 跨区块边界采用显式交接账本
- 当目标区块不可写时，流量进入 pending handoff，而非直接丢弃或伪写。
- 目标区块加载后按时序回放交接流量。

Rationale:
- 消除 chunk 边界悬滴与守恒破坏。

### Decision 5: 渲染层只消费增量脏区并对齐阈值合同
- 渲染不参与物理决策。
- 可视阈值与逻辑连通阈值建立一致性窗口。
- 采用增量 dirty 更新，避免全量复制。

Rationale:
- 降低帧耗并减少“逻辑有水、画面断裂”的错觉。

## Alternatives Considered
- 继续在现有系统上增量修补：
  - 优点：短期改动小。
  - 缺点：规则继续耦合，回归风险继续累积。
- 采用全连续流体（Navier-Stokes / SPH）替代网格：
  - 优点：物理更真实。
  - 缺点：与当前 tile/chunk 架构不匹配，性能和工程成本不可控。

## Risks / Trade-offs
- 风险：重构规模大，短期内回归面广。
  - 缓解：并行双路径验证与分阶段切换。
- 风险：固定精度会降低细腻度。
  - 缓解：渲染层连续化与局部亚格表现，不牺牲权威状态稳定性。
- 风险：边界交接账本增加状态复杂度。
  - 缓解：定义严格生命周期与守恒断言，加入专项回归。

## Migration Plan
1. 建立新数据模型与守恒断言，不改默认求解路径。
2. 引入新求解器并与旧求解器并行采样比对。
3. 引入新调度器和跨区块交接账本。
4. 切换渲染为增量脏区与阈值一致性合同。
5. 完成性能回归后切换默认，保留回滚开关。

## Validation Strategy
- 功能验证：
  - 空泡收敛时间阈值
  - 悬滴残留率阈值
  - 跨区块连续下泄
- 正确性验证：
  - 单 tick 守恒
  - save/load/unload 端到端守恒
- 性能验证：
  - P50/P95/P99 帧耗
  - 活跃单元吞吐
  - 调度器饥饿率
- 确定性验证：
  - 固定种子同输入同输出

## Reference Notes

- W-Shadow / Gallant 的 2D CA 液体实践：稳定状态分配和本地规则。
- Noita 公开架构方向：大世界中基于栅格物理可实现且有玩法价值。
