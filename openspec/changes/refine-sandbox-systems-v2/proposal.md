# Change: 深度细化 14 大沙盒核心系统 (Refinement)

## Why
在初步建立了 MVP 脚本骨架后，需要根据用户对“挖掘自动拾取”、“装备继承”、“工业电路”等具体细节的要求，进行深度的功能拆解与任务细化。这确保了后续开发不仅有逻辑，还能满足玩家的可视化交互体验。

## What Changes
- **系统深度扩展**：
    - **挖掘系统**：从简单的“采集”升级为基于 TileMap 的地形破坏与掉落物自动吸附。
    - **血脉系统**：细化“转生”逻辑，确保属性继承与装备保留的准确性。
    - **工业系统**：定义基础电路逻辑（逻辑门、电源、导线）。
    - **战斗系统**：增加弹药消耗与多层图层切换的视觉反馈。
- **任务细化**：将 14 个大系统拆分为 50+ 个可验证的小任务。
- **交互增强**：所有系统必须预留 UI 钩子，确保“全可视化”目标的达成。

## Collaboration Model
- **我（脚本/逻辑）**：编写复杂的系统逻辑（如 TileMap 挖掘算法、遗传算法、电路模拟）。
- **你（编辑器/视觉）**：配置 TileSet（设置哪些层可挖掘）、搭建复杂的 UI 界面、配置 NPC 动画状态机。

## Impact
- **Affected specs**: 覆盖所有 14 个核心 capability 的深度扩展。
- **Affected code**: `res://src/systems/` 下的所有模块。
